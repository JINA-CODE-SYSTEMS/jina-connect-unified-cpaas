## Summary

First-class RCS Business Messaging channel adapter with Google RBM provider, rich card + carousel builder, capability checks, SMS fallback, and iOS-optimised rendering.

**Full plan:** `RCS_IMPLEMENTATION_PLAN.md` (2300+ lines) — see `rcs-channel` branch.
**Depends on:** #28 (Google RCS adapter spec), #29 (Meta RCS adapter spec)

## Scope — Phase 1 (Core Channel Plumbing)

### New Django App: `rcs/`

- **Models:** `RCSApp` (provider credentials encrypted via `FIELD_ENCRYPTION_KEY`), `RCSWebhookEvent`, `RCSOutboundMessage`
- **Provider layer:** `BaseRCSProvider` ABC -> `GoogleRBMProvider` (OAuth2 service account, REST API)
- **Rich content builders:** `RichCardBuilder` (standalone + carousel, 3:2 iOS safe area), `SuggestionBuilder` (replies, actions — dial/map/URL/calendar)
- **Capability checker:** `RCSCapabilityChecker` — per-user + batch, Redis cache, device OS detection (Android vs iOS)
- **Message sender:** `RCSMessageSender` implements `BaseChannelAdapter` (send_text, send_media, send_keyboard + RCS-specific send_rich_card, send_carousel)
- **iOS-aware rendering:** `_detect_device_os()` + `_adjust_for_ios()` — auto 3:2 media ratio, <=20 char suggestion text, carousel fallback for pre-18.2
- **SMS fallback:** Auto-fallback to SMS via `sms_fallback_app` when capability check returns 404. Daily counter decremented on fallback (not consumed on failed RCS).
- **Daily counter:** Atomic `increment_daily_counter()` with `__lt=F('daily_limit')` guard (mirrors SMS pattern)
- **Channel registration:** Register `'RCS'` in `channel_registry` via `RCSConfig.ready()`
- **Webhook receiver:** `POST /rcs/v1/webhooks/<uuid>/` — Google Pub/Sub push, `X-Goog-Signature` HMAC-SHA512 validation
- **Celery task:** `process_rcs_event_task` — inbound processing -> contact upsert + inbox message + DLR/read updates
- **Rate limiter:** Redis-backed per-tenant throttling (`PLATFORM_RATE_LIMITS['rcs']` = 300/min)

### Security

- `provider_credentials` encrypted at rest (EncryptedTextField)
- `X-Goog-Signature` HMAC-SHA512 webhook validation
- Tenant isolation on all queries
- OAuth2 tokens cached 55 min, never stored in DB
- Silent webhook responses prevent information leakage

### Architecture Reference

Mirrors Telegram (PR #77) + SMS (#80) adapter patterns. Reuses:
- `wa/adapters/channel_base.py` -> `BaseChannelAdapter`
- `jina_connect/channel_registry.py` -> `register_channel()` / `get_channel_adapter()`
- `team_inbox/utils/inbox_message_factory.py` -> `create_inbox_message()`
- `abstract/models.py` -> `BaseWebhookDumps`
